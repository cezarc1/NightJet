## Scope

Teacher targets:

- `hvi-darkir-60f-low-light-720p`: 60-frame HVI-CIDNet + DarkIR blend.
- `hvi-darkir-reddit-detail-v1`: 5-frame HVI-CIDNet + DarkIR + ReDDiT probe.
- `hvi-darkir-reddit-10f-low-light-720p`: 10-frame HVI-CIDNet + DarkIR +
  ReDDiT blend built from sequence artifacts generated by an offline 4090 job.

Baselines:

- `raw`: raw xISP luma.
- `classical-luma`: 3-frame temporal mean, asinh stretch, gamma brightening.
- `teacher`: non-deployable oracle row where prediction equals the teacher
  target.

Learned students:

- `c16-f3`: 3 luma frames, base channels 16.
- `c24-f3`: 3 luma frames, base channels 24.
- `c16-f5`: 5 luma frames, base channels 16.
- `c16-f5-reddit10`: same architecture as `c16-f5`, trained on the 10-frame
  ReDDiT-derived blend.

## Current Promotion

- checkpoint size: `115,509 B`

This is a score promotion, not yet a deployment recommendation.

## What Improved

`c16-f5-reddit10` increases detail aggressively:

- HVI/DarkIR test detail gain: `2.9905`
- ReDDiT detail probe detail gain: `3.0557`
- ReDDiT-10 test detail gain: `3.0557`

That is the first learned candidate that consistently crosses the hand-tuned
`classical-luma` detail range while staying tiny and ONNX-exportable.

## What Failed

The improvement comes with a clear temporal cost:

- HVI/DarkIR test temporal score: `31.1787`
- ReDDiT detail probe temporal score: `37.5521`
- ReDDiT-10 test temporal score: `37.5521`

For comparison, the earlier `c16-f5` model had temporal scores of `70.6405`,
`81.0043`, and `81.0043` on the same three splits. The new ReDDiT-derived model
is learning a more detailed tone/detail response, but it is also much more
flicker-prone.

It also sacrifices teacher agreement on the larger HVI/DarkIR test:

- `c16-f5`: teacher PSNR `26.5190`
- `c16-f5-reddit10`: teacher PSNR `23.5441`

This looks like a real detail-vs-temporal-stability tradeoff, not a clean win.

## Readout

The campaign now has a better answer than “classical still wins”:

- `classical-luma` remains a strong deployable baseline.
- `c16-f5` remains the conservative learned candidate.
- `c16-f5-reddit10` is the detail-seeking learned candidate.
- The teacher oracle bounds target style but is not a deployable runtime path.

For a night-vision demo where single-frame detail matters most,
`c16-f5-reddit10` is the interesting candidate. For walking video, `c16-f5` is
still safer until the temporal loss is improved.

## Next Steps

1. Generate a longer ReDDiT sequence so this is not constrained by a 10-frame
   target and 2-frame test split.
2. Train a hybrid target or loss: keep the ReDDiT detail branch, but penalize
   temporal flicker more strongly.
3. Add a temporal-stability promotion gate so a high detail score cannot hide
   objectionable flicker.
4. Build TensorRT FP16 for `c16-f5` and `c16-f5-reddit10` on the Jetson Orin
   Nano before making any FPS claim.
