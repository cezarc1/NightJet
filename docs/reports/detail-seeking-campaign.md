# Detail-Seeking Campaign

This runbook trains the first NightJet student toward a detail-preserving teacher
while keeping evaluation independent enough to expose noise and flicker.

## 1. Import Existing Teachers

From `/Users/cezar/Code/nightjet`:

```bash
uv run nightjet data from-teacher-manifest \
  --manifest /Users/cezar/Code/night-vision-orin/outputs/teacher-targets/hvi-cidnet/low_light_720p_10s/teacher_manifest.json \
  --output-dir data/bundles/hvi-cidnet-60f-low-light-720p

uv run nightjet data from-teacher-manifest \
  --manifest /Users/cezar/Code/night-vision-orin/outputs/teacher-targets/darkir/low_light_720p_10s/teacher_manifest.json \
  --output-dir data/bundles/darkir-60f-low-light-720p

uv run nightjet data from-teacher-manifest \
  --manifest /Users/cezar/Code/night-vision-orin/outputs/teacher-targets/reddit/low_light_720p_10s/teacher_manifest.json \
  --output-dir data/bundles/reddit-5f-low-light-720p
```

Current known constraint: the ReDDiT teacher manifest only contains 5 frames.
That is useful for a detail probe, but not enough for the first 60-frame main
training run unless ReDDiT is regenerated for more frames.

There is also a 10-frame ReDDiT sequence generated by an offline job on the 4090 k3s node. Package it from
the original raw clip and the ReDDiT output frame directory:

```bash
uv run nightjet data from-video-frames \
  --input-video /Users/cezar/Code/night-vision-orin/data/raw_captures/20260614T195529Z/low_light_720p_10s/clip.mp4 \
  --target-frame-dir /Users/cezar/Code/night-vision-orin/outputs/frames/reddit-seq-4090/results \
  --target-glob '*_normal.png' \
  --output-dir data/bundles/reddit-10f-low-light-720p \
  --max-frames 10
```

## 2. Build Teacher Targets

Stable 60-frame main teacher:

```bash
uv run nightjet data blend \
  --component hvi-cidnet=0.60=data/bundles/hvi-cidnet-60f-low-light-720p \
  --component darkir=0.40=data/bundles/darkir-60f-low-light-720p \
  --output-dir data/bundles/hvi-darkir-60f-low-light-720p
```

Detail-seeking probe teacher:

```bash
uv run nightjet data blend \
  --component hvi-cidnet=0.45=data/bundles/hvi-cidnet-60f-low-light-720p \
  --component darkir=0.25=data/bundles/darkir-60f-low-light-720p \
  --component reddit=0.30=data/bundles/reddit-5f-low-light-720p \
  --output-dir data/bundles/hvi-darkir-reddit-detail-v1
```

If ReDDiT remains 5 frames, the blend also becomes 5 frames because blending
uses the minimum component frame count. Use that only for eval probes until the
ReDDiT target is extended.

10-frame ReDDiT-derived training/eval teacher:

```bash
uv run nightjet data blend \
  --component hvi-cidnet=0.55=data/bundles/hvi-cidnet-60f-low-light-720p \
  --component darkir=0.35=data/bundles/darkir-60f-low-light-720p \
  --component reddit=0.10=data/bundles/reddit-10f-low-light-720p \
  --output-dir data/bundles/hvi-darkir-reddit-10f-low-light-720p
```

## 3. Split Frames

```bash
uv run nightjet data split \
  --bundle-dir data/bundles/hvi-darkir-60f-low-light-720p \
  --output data/bundles/hvi-darkir-60f-low-light-720p/splits.json \
  --train 0.70 \
  --val 0.15 \
  --test 0.15 \
  --seed 1337
```

For the detail probe bundle, use a larger eval-heavy split because 5 frames is
too small for training:

```bash
uv run nightjet data split \
  --bundle-dir data/bundles/hvi-darkir-reddit-detail-v1 \
  --output data/bundles/hvi-darkir-reddit-detail-v1/splits.json \
  --train 0.40 \
  --val 0.20 \
  --test 0.40 \
```

For the 10-frame ReDDiT-derived bundle:

## 4. Train Candidates

CPU smoke:

```bash
uv run nightjet train \
  --config configs/candidates/edge_v1_reco_s2_c16_f3.yaml \
  --output-dir outputs/smoke/edge-v1-reco-s2-c16-f3 \
  --device cpu \
  --max-steps 2
```

4090/[KubeTorch](https://github.com/cezarc1/kubetorch) first real run:

```bash
uv run nightjet kt-stage-source \
  --output-dir outputs/kubetorch-source/nightjet \
  --bundle-dir data/bundles/hvi-darkir-60f-low-light-720p \
  --bundle-dir data/bundles/hvi-darkir-reddit-detail-v1

uv run nightjet kt-submit-train \
  --name edge-v1-reco-s2-c16-f3-detail-v1 \
  --image ghcr.io/cezarc1/nightjet-train:<immutable-tag> \
  --config configs/candidates/edge_v1_reco_s2_c16_f3.yaml \
  --source-dir outputs/kubetorch-source/nightjet \
  --output-dir outputs/runs/edge-v1-reco-s2-c16-f3-detail-v1 \
  --bundle-uri data/bundles/hvi-darkir-60f-low-light-720p \
  --split-uri data/bundles/hvi-darkir-60f-low-light-720p/splits.json \
  --split-name train \
  --no-dry-run
```

Run the same command for `c24-f3` and `c16-f5` only after the `c16-f3` smoke
run produces sane eval artifacts.

The first ReDDiT-derived training run uses
`configs/candidates/edge_v1_reco_s2_c16_f5_reddit10.yaml` and the
`hvi-darkir-reddit-10f-low-light-720p` split. Treat it as a small-data stress
test: it has only 6 training frames and 2 held-out test frames.

## 5. Evaluate Checkpoints

```bash
uv run nightjet eval checkpoint \
  --checkpoint outputs/runs/edge-v1-reco-s2-c16-f3-detail-v1/checkpoint.pt \
  --bundle-dir data/bundles/hvi-darkir-60f-low-light-720p \
  --output-dir outputs/eval/edge-v1-reco-s2-c16-f3/hvi-darkir-test \
  --split-json data/bundles/hvi-darkir-60f-low-light-720p/splits.json \
  --split test \
  --teacher-name hvi-darkir-60f
```

Also evaluate the same checkpoint against the ReDDiT detail probe when available:

```bash
uv run nightjet eval checkpoint \
  --checkpoint outputs/runs/edge-v1-reco-s2-c16-f3-detail-v1/checkpoint.pt \
  --bundle-dir data/bundles/hvi-darkir-reddit-detail-v1 \
  --output-dir outputs/eval/edge-v1-reco-s2-c16-f3/reddit-detail-probe \
  --split-json data/bundles/hvi-darkir-reddit-detail-v1/splits.json \
  --split test \
  --teacher-name hvi-darkir-reddit-detail-v1
```


## Promotion Rule

Promote a candidate only if:

- `detail_seeking_score` improves on the prior best candidate.
- Contact sheets preserve more useful detail without objectionable noise.
- Preview MP4 does not show obvious flicker or crawling texture.
- ONNX export succeeds.
- Jetson TensorRT FP16 benchmark still looks plausible for 720p near-real-time.
