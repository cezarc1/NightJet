# NightJet Model Architecture

NightJet is an edge low-light enhancement project, not a detector, classifier,
LLM, or diffusion model. The public model is a tiny temporal luma enhancer:
given a short window of low-light luma frames, it predicts a bounded correction
to the latest frame.

The design goal is practical Jetson deployment. A useful model has to be small
enough for the 8 GB Jetson Orin Nano path, export cleanly to ONNX, and build as
a TensorRT FP16 engine on the device.

## Naming

The names are intentionally separated:

| Name | Meaning |
| --- | --- |
| `NightJet` | This public inference, training, evaluation, export, and [KubeTorch](https://github.com/cezarc1/kubetorch) submission repo. |
| `NightJetEdgeV1` | The current candidate architecture implemented in this repo. |
| `nightjet-edge-v1` | The default public weight in `weights/nightjet-edge-v1.pt`. |
| `nightjet-edge-v1-detail` | The sharper public variant in `weights/nightjet-edge-v1-detail.pt`. |
| `night-vision-orin` | Advanced Jetson camera/runtime companion, not required for image/video inference. |
| [`KubeJet`](https://github.com/cezarc1/kubejet) | Optional k3s/Kubernetes operator and `EdgeVisionPipeline` deployment path. |
| [`KubeTorch`](https://github.com/cezarc1/kubetorch) | Optional Kubernetes batch-run path used to train and evaluate candidates on the homelab GPU node. |

`NightJetEdgeV1` candidates are score-promoted in this repo before they are
runtime-promoted. A public PyTorch/ONNX artifact is still not a portable
TensorRT deployment until the target Jetson engine is built and validated.

## Data Flow

```text
Arducam UYVY low-light clips
  -> luma frame bundles
  -> teacher-model output bundles
  -> NightJetEdgeV1 student training
  -> checkpoint.pt + run/eval artifacts
  -> public inference-only .pt + ONNX export
  -> optional Jetson-built TensorRT FP16 engine
  -> optional night-vision-orin or KubeJet deployment
```

NightJet keeps public image/video inference and the model-development loop here.
Camera bring-up, TensorRT serving, MJPEG streaming, and live
[KubeJet](https://github.com/cezarc1/kubejet) deployment belong in advanced
companion repos.

## NightJetEdgeV1

`NightJetEdgeV1` accepts luma tensors shaped:

```text
B x T x H x W
```

where `T` is usually 3 or 5 frames. The current output is a single enhanced
luma frame:

```text
B x 1 x H x W
```

The architecture has three small parts:

| Part | Role |
| --- | --- |
| Coarse trunk | Processes the full temporal luma window at a lower spatial scale when `trunk_scale > 1`. |
| Detail branch | Processes the full-resolution temporal luma window with shallow convolutions. |
| Residual fuse | Concatenates the latest frame, trunk features, and detail features, then predicts a bounded residual. |

The residual form is:

```text
output = clamp(latest_frame + residual_scale * tanh(fuse(features)), 0, 1)
```

That keeps the model conservative: it learns a correction to the camera feed
instead of generating an image from scratch.

## Current Public Shapes

The public v0.1 weights use deliberately small 5-frame candidates:

| Config | Frames | Base channels | Detail channels | Parameters | Notes |
| --- | ---: | ---: | ---: | ---: | --- |
| `nightjet-edge-v1` | 5 | 16 | 8 | 2,778 | Conservative default. |
| `nightjet-edge-v1-detail` | 5 | 16 | 8 | 2,778 | Sharper detail variant with higher flicker risk. |

The exact public artifact contract is in `weights/manifest.json`. Training
candidate settings remain in `configs/` and `configs/candidates/`.

## Training Objective

NightJet is trained as a student model. The inputs are real low-light luma frame
windows, and the targets are outputs from stronger enhancement models. Some
bundles are generated by jobs on the 4090 k3s node, but the 4090 is compute
provenance, not the teacher. Current teacher bundles blend or compare
HVI-CIDNet, DarkIR, and ReDDiT-style detail targets.

The current loss family is simple and inspectable:

| Loss | Purpose |
| --- | --- |
| Reconstruction | Match the teacher target. |
| Edge/detail | Reward useful local detail. |
| Brightness | Avoid under-enhanced dark output. |
| Smoothness | Penalize temporal flicker in multi-frame candidates. |

This is closer to supervised video-to-video regression than to open-ended
generative restoration.

## Why This Shape

The model is luma-first because the camera/runtime path already exposes UYVY
frames and the first deployment target is a passive low-light visibility demo.
Luma enhancement also keeps the training target narrower than full RGB
restoration.

The network is convolutional because the target runtime is small, fixed-shape,
and TensorRT-friendly. The temporal window adds a little motion context without
requiring an expensive video transformer.

The residual head keeps failure modes more bounded than a direct image generator:
bad predictions are corrections on top of the raw feed, not an entirely
hallucinated frame.

## Current Limits

- This is a luma enhancement path, not a full-color restoration model.
- It does not make truly absent light recoverable.
- Detail-seeking candidates can introduce temporal flicker.
- Public ONNX promotion in this repo is not the same as Jetson TensorRT
  deployment promotion.
- Final FPS and latency claims must be measured on the Jetson Orin Nano target,
  not on the 4090 k3s training node.
